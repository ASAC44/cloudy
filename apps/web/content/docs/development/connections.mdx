---
title: Service connections
description: Connect Linear, Stripe, OAuth providers, and trusted remote MCP servers safely.
---

Podex stores connection credentials encrypted with `CONNECTION_ENCRYPTION_KEY`.
The dashboard never receives a saved key after submission, and replacing a key
runs a fresh connection test.

## Connect Linear

1. In Linear, open **Settings → Account → Security & Access** and create an API
   key for Podex. Prefer the narrowest access available for the workspace and
   actions you plan to use.
2. In Podex, open **Connections**, choose **Linear**, name the connection, and
   paste the key.
3. Select **Save & test**. Podex connects only to Linear's official Streamable
   HTTP endpoint at `https://mcp.linear.app/mcp` and reports the number of tools
   it discovers.

See [Linear's MCP documentation](https://linear.app/docs/mcp) for current key
and server requirements.

## Connect Stripe

Start in a Stripe sandbox. In **Developers → API keys**, create a restricted
key that grants only the resources and operations required by the Ping. Do not
use a publishable key, and avoid unrestricted or live keys during setup.

Choose **Stripe** on the Podex Connections page and paste the restricted key.
Podex connects only to `https://mcp.stripe.com`. Stripe controls which tools are
available from the permissions on that key. See [Stripe's MCP documentation](https://docs.stripe.com/mcp)
and [key security guidance](https://docs.stripe.com/keys-best-practices).

## Capability safety

Podex follows MCP tool annotations instead of guessing what a tool does:

- Tools explicitly marked read-only may provide source or context data.
- Tools explicitly marked non-destructive writes may become rule actions, but
  the exact payload must be approved on the Pod before execution.
- Destructive tools are excluded.
- Unannotated tools are shown as unavailable at runtime.

Provider permissions remain an independent safety boundary. Give each key only
the access its rule needs even when Pod approval is enabled.

## Rotate or revoke a key

Use **Replace token** on the connection, paste the new key, and let the smoke
test finish. Then revoke the old key in Linear or Stripe. If a key might have
been exposed, revoke it at the provider first and replace it in Podex
immediately. Disconnecting a provider deletes its encrypted credential from
Podex but does not revoke the key at the provider.

GitHub and Gmail continue to use OAuth. Vercel and Telegram use their existing
token or account flows. **Custom MCP** remains available for other public HTTPS
Streamable HTTP endpoints.
