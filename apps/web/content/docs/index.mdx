Podex gives AI agents and automated workflows a physical decision point. An agent proposes an action, the Pod shows the exact context that matters, and a person approves, rejects, or revises it.

## System boundaries

The Next.js service hosts this documentation and the dashboard. The Hono API owns authentication checks and state-changing endpoints, and its guarded runtime loop handles dynamic-rule polling, evaluation, approved delivery, callbacks, and Telegram sessions. Supabase is the canonical data store. The Pod holds only its scoped credential and the active request needed for offline reading.

Podex never treats model output as authorization. Approval stays bound to the exact action and payload shown to the user.
